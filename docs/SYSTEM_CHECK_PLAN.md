## 점검 방법

### 자동 검증
```bash
# 1. Import 검증
python scripts/verify_imports.py

# 2. 기본 기능 테스트 (작성 예정)
python scripts/system_health_check.py
```

### 수동 검증
1. **Streamlit 앱 실행**
   ```bash
   streamlit run src/ui/app.py
   ```

2. **각 탭별 순차 테스트**
   - 병리 이미지 분석
   - CT/MRI 분석
   - 배치 처리
   - 분석 인사이트

3. **샘플 데이터 사용**
   - 테스트 이미지 준비
   - DICOM 파일 준비
   - 환자 데이터 샘플

## 문제 발견 시 처리

### Critical (즉시 수정)
- 앱 크래시
- 핵심 기능 완전 불가
- 데이터 손실

### High (배포 전 수정)
- 주요 기능 오작동
- 명확한 에러 메시지
- UI 심각한 문제

### Medium (배포 후 개선)
- 성능 이슈
- UI 불편함
- 경고 메시지

### Low (추후 개선)
- 코드 최적화
- 문서 보완
- 추가 기능

## 점검 일정

### 즉시 (30분-1시간)
1. Streamlit 앱 실행 및 전체 탭 확인
2. 병리 이미지 분석 재테스트
3. 주요 에러 확인

### 단기 (2-3시간)
1. CT/MRI 기능 점검
2. CDSS 기능 점검
3. 발견된 문제 수정

### 중기 (1일)
1. 전체 기능 통합 테스트
2. 샘플 데이터로 E2E 테스트
3. 문서 최종 정리

## 점검 결과 문서

각 점검 항목별로:
- ✅ 정상 작동
- ⚠️ 경미한 문제 (동작 가능)
- ❌ 심각한 문제 (수정 필요)
- ⏳ 미점검

결과는 `SYSTEM_CHECK_REPORT.md`에 기록
